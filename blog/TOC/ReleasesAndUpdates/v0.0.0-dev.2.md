This release removes the dependency on **@fizzwiz/sorted** to **avoid circular dependencies**, since the next version of `@fizzwiz/sorted` will depend on `@fizzwiz/prism`. This keeps both libraries modular, maintainable, and free from interlocking build issues.

### What Changed

The `Search` and `AsyncSearch` constructors **no longer instantiate an `ArrayQueue` by default**. You now need to explicitly pass a queue instance using the `via()` semantic:

```js
const search = new Search()
  .from(start)
  .through(space)
  .via(queue);
```

This change gives you finer control over the search behavior while ensuring Prism stays lightweight and dependency-free.
